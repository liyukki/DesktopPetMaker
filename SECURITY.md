# 安全说明

## 报告问题

请不要在公开 Issue 中提交真实 API Key、Bearer Token、Credential Store 内容、用户聊天记录或包含个人路径的完整日志。

报告安全问题时请提供：

- 受影响版本
- 最小复现步骤
- 预期行为与实际行为
- 已脱敏的日志
- 可能影响的数据范围

## 凭据处理

- API 凭据只应保存在运行时凭据存储中。
- `pet.json`、`.petpack`、截图、报告和 Review Bundle 中不得包含真实凭据。
- 如果凭据曾出现在截图或提交历史中，应立即撤销并重新生成。

## 当前边界

项目仍处于 Beta 阶段。公开发布前应完成干净 Windows 环境测试、依赖许可证确认、二进制签名和最终 Secret 扫描。
